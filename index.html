Blogging with Hugo

So this is my first post, and this is the one that is serving me (have served) for testing. How did I come to write a blog is out of the question, this starts at the point when that decision was already taken.

This blog needs to conform to this requisites:

  • Cheap
  • No ads
  • Custom domain
  • Allow visitors to drop a line
  • Customizable
  • Simple editing

Requisites

Cheap

There are some well known places to host a blog for free, but they don’t usually allow custom domain names and customizing. I don’t think I’ll want to use the server/service for other matters, so I’m not willing to spend over 50€/year, including domain name.

Ads

Blogspot, Wordpress and the likes can be used without ads, and obviously if you pay for a service you are getting rid of them. Some other services like blog.com, include ads in your posts when using the free version.

Custom domain

I’m looking for a registrar that allows me to define alias to redirect to wherever my site is. Not all registrars sell every TLD, so chose an available domain and filtered companies that didn’t offered ‘.me’ domains (which is my surname prefix). ‘.me’ domain’s prices are lower than most of the other TLDs, and although pablo.me would have been better, pabl.me is ok for me.

Comments

I’ve already used Wordpress and Joomla before, and although having an full featured solution for the blog (with comments, images, videos, logs …) seems right, there are some other products that can be integrated seamlessly into the blog post’s.

Customization

I’m far from being a web designer, so I can relax customization, as far as I can use some already made theme and tweak it a little bit. I don’t like Blogspot appearance but probably anything beyond Blogspot is going to be OK for me.

Simple editing

No HTML. I’d like to use a simple editor that allows inserting external objects and custom formating for code. Wordpress have some amazing plugins for that.

The hosting

Let’s start looking at the three possible blog hosting families I’m thinking of:

  • Small cloud server: mmm … not price compliant with the requisites.
  • A web hosting service: if used this one before, and feel confortable.
  • A blog service: should be cheaper than the web hosting.

So let’s begin dropping the cloud server, using a go/node/ruby application and extending the server with some other applications is tempting, but I don’t want to lose focus. I’ll try the blog service solution, and if I find it doesn’t suits me, I’ll fall back to the web hosting service.

After some googling, my two main alternatives are

  • Use AWS services, not EC2
  • Use Github pages

JAWS framework have been around for a while. At this point in time it’s about to release v1.0.0. JAWS is using mainly S3 for hosting and serving all static content, Lambda to serve requests that require processing, and as far as I know it uses dynamo to store “alive” data.

JAWS is great and cheap, If I were starting a business on a budget I would surely take this way, but for a blog seems huge. Once more I have to stay focused, I only need some static content, and S3 allows me to host that static content and also to use my own domain.

S3 price is low, way low, I can use whatever tool to generate static content, bring my domain, and pay only for storage and network usage, which will propably be under 1€ (I’m not expecting many visitors). But … Github is free.

If you don’t mind having all you files public, and I don’t mind, Github can host our website. After reading a bit I decided to get the Github way, so my only expense is going to be the domain and my own time.

Domain

After some googling for registrars, Namecheap seemed to be good choice for me, so I looked for a discount coupon using Honey extension for Chrome, and purchased it. I also followed Github Pages instructions to configure the A record in Namecheap control panel. In a few minutes I had a Hello World ugly page under my domain.

The Blog

I still need to read Github Pages documentation and whatever differences are there between static web generators, Jekyll, Hugo, Octopress, Middleman to name a few.

I’ll stick with Hugo because:

  1. It’s written in Go, and I love Go.
  2. One of my sons is called Hugo.
  3. I don’t think spending much time comparing static web generators will make a difference.

I guess this three reasons sound a bit lazy, but hey, I wan’t my blog now.

Si I downloaded Hugo, read (some of) the docs, and started to Lorem ipsum with it. I cloned all Hugo themes and tested each one of them. I liked purehugo as a starting point, it’s very simple yet is integrated with Disqus for comments.

There are some little flaws that I’m fixing, and a couple of features I’d like to have that are missing. One of them is showing only the first few words of a post in listings, instead of showing the full post. That’s suppported in Hugo using the ‘Summary’ variable.

That one is going to serve me to test the source code rendering with Hugo. Let’s see the theme’s layouts/_default/lists.html file modified, replacing the content div with:

<div class="summary">
    {{ .Summary }}
</div>
{{ if .Truncated }}
<div class="read-more-link">
  <a href="{{ .RelPermalink }}">Read More…</a>
</div>
{{ end }}
